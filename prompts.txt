PROMPT:
/speckit-plan Use Option A (Synchronous Atomic Increments) described in
@docs/ as the definitive architecture — do not re-evaluate options B/C.

V1 Stack:
- REST/JSON API in Go using net/http + chi; OpenAPI specification for
  web/mobile clients.
- PostgreSQL via pgx + sqlc; migrations managed with golang-migrate (executed as
  an init container or Docker Compose command).
- Data model according to section 1.5 of research.md (users,
  feature_requests, votes with a composite primary key, indexes idx_fr_top,
  idx_fr_created, idx_votes_request).
- Voting: single transaction (INSERT ... ON CONFLICT DO NOTHING +
  UPDATE vote_count if the insert affected a row).
- Redis: count cache + "top" and "trending" ZSETs
  (score = vote_count / (age_hours + 2)^1.5, recomputed by a periodic job
  every 1–5 minutes).
- Authentication: JWT (access + refresh), argon2id.
- Cursor-based pagination + ETag/304 for list endpoints.
- Observability: JSON slog, /healthz, /readyz, /metrics (Prometheus RED),
  OpenTelemetry traces.
- Testing: unit tests, integration tests with testcontainers (real PostgreSQL + Redis),
  and end-to-end tests using httptest.
- Deployment: multi-stage Dockerfile (builder -> distroless), docker-compose.yml
  with app/postgres/redis services, health checks, depends_on, named volumes,
  and .env/env_file support.

Check, Project Structure), research.md, data-model.md, contracts/openapi.yaml,
